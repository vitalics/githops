# Обновление

## Самообновление

Самый простой способ обновить githops — встроенная команда самообновления:

```sh
githops self-update
```

Она скачивает последний релиз для вашей платформы с GitHub, заменяет текущий бинарный файл и выводит новую версию.

Чтобы проверить наличие обновлений без установки:

```sh
githops self-update --check
```

## Обновление через менеджеры пакетов

### Cargo

```sh
cargo install githops --force
```

### npm / pnpm

```sh
# npm
npm install --save-dev githops@latest

# pnpm
pnpm add --save-dev githops@latest
```

### macOS PKG / DMG

Скачайте последний `.pkg`-установщик со [страницы релизов](https://github.com/vitalics/githops/releases/latest) и запустите его. Установщик обнаруживает существующую установку через `UpgradeCode`, встроенный в пакет, и обновляет на месте.

### Windows MSI

Скачайте последний `.msi` со [страницы релизов](https://github.com/vitalics/githops/releases/latest) и запустите. MSI-установщик автоматически обрабатывает обновление.

## Совместимость конфигурации

githops следует семантическому версионированию. Минорные и патч-релизы обратно совместимы — существующий `githops.yaml` продолжит работать после обновления.

Мажорные версии могут содержать ломающие изменения. При выходе мажорной версии в журнале изменений релиза публикуется заметка о миграции с описанием изменений и инструкциями по обновлению конфигурации.

## Обновление JSON Schema

После обновления пересгенерируйте файл схемы для получения IntelliSense для новых полей:

```sh
githops init
```

Это записывает только `.githops/githops.schema.json` — `githops.yaml` не изменяется.

## Повторная синхронизация хуков после обновления

Скрипты хуков, записанные в `.git/hooks/` командой `githops sync`, делегируют выполнение бинарному файлу `githops`. В большинстве случаев повторная синхронизация после обновления не нужна, поскольку скрипты просто вызывают `githops run <имя-хука>` и используют тот бинарный файл, который находится в `PATH`.

Если мажорная версия изменяет интерфейс исполнителя хуков, примечания к релизу об этом сообщат и включат инструкции по повторной синхронизации.

## Фиксация версии

Если нужна конкретная версия githops в CI-окружении или вы хотите избежать автоматических обновлений:

```sh
# Установка конкретной версии через cargo
cargo install githops --version 1.2.3 --force

# Или через npm
npm install --save-dev githops@1.2.3
```

## Журнал изменений

Полный журнал изменений доступен на [странице релизов GitHub](https://github.com/vitalics/githops/releases). Каждый релиз включает список коммитов и инструкции по установке.
