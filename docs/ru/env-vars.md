# Переменные окружения

## Переменные, читаемые githops

| Переменная | Используется в             | Назначение                                                                  |
| ---------- | -------------------------- | --------------------------------------------------------------------------- |
| `SHELL`    | `githops completions init` | Определяет текущую оболочку для выбора формата дополнений и пути установки. |
| `HOME`     | `githops completions`      | Разворачивает `~` при записи файлов дополнений и патчинге RC-файлов.        |

## Переменные в командах хуков

Любая команда в `githops.yaml` может объявить блок `env:`. githops передаёт эти переменные в подпроцесс при выполнении команды:

```yaml
hooks:
  pre-commit:
    commands:
      - name: lint
        run: eslint .
        env:
          NODE_ENV: production
          CI: "true"
```

Переменные ограничены конкретной командой и не попадают в другие команды и в сам процесс githops.

Команда `$include` также поддерживает `env:`:

```yaml
hooks:
  pre-commit:
    commands:
      - $include: packagejson
        run: scripts.lint
        args: "--fix"
        env:
          NODE_ENV: production
```

## Переменные, переданные из git

Когда git вызывает хук, он передаёт аргументы, доступные в командах через `$1`, `$2` и т.д. Для `commit-msg` git передаёт путь к файлу сообщения коммита как `$1`:

```yaml
hooks:
  commit-msg:
    commands:
      - name: validate
        run: node scripts/validate-msg.js $1
```

## Подробное логирование (verbose)

Два параметра командной строки управляют структурированным выводом в stderr:

| Параметр             | По умолчанию        | Описание                                                                 |
| -------------------- | ------------------- | ------------------------------------------------------------------------ |
| `-v`, `--verbose`    | выкл.               | Включить подробное логирование. Без флага выводятся только INFO и ERROR. |
| `--verbose-template` | `[$t] [$k] ($l) $m` | Формат каждой строки лога (см. ниже).                                    |

### Виды (kind)

| Kind      | Когда выводится                                                                   |
| --------- | --------------------------------------------------------------------------------- |
| `INFO`    | Всегда (даже без `-v`). Ключевые события (конфиг загружен, хук выполнен).         |
| `VERBOSE` | Только с `-v`. Детали операций (разрешение команд, запуск каждой команды).        |
| `ERROR`   | Всегда. Ошибки до аварийного завершения.                                          |
| `TRACE`   | Только с `-v`. Внутренние детали (попадания в кэш, строки exec, количество волн). |

### Слои (layer)

| Layer               | Подсистема                                                                |
| ------------------- | ------------------------------------------------------------------------- |
| `schema validation` | Генерация и валидация JSON-схемы (`githops schema sync`, `githops sync`). |
| `yaml resolve`      | Загрузка конфигурации и разрешение includes.                              |
| `yaml exec`         | Выполнение команд хуков (`githops check`).                                |

### Токены шаблона

| Токен | Заменяется на                                            |
| ----- | -------------------------------------------------------- |
| `$t`  | Время — `ЧЧ:ММ:СС.ммм` (UTC)                             |
| `$k`  | Kind — `INFO`, `VERBOSE`, `ERROR`, `TRACE`               |
| `$l`  | Layer — `schema validation`, `yaml resolve`, `yaml exec` |
| `$m`  | Текст сообщения                                          |

### Примеры

Вывод по умолчанию:

```
[12:34:56.789] [INFO] (yaml resolve) config loaded from githops.yaml
[12:34:56.791] [VERBOSE] (yaml resolve) resolved 3 command(s)
[12:34:56.792] [TRACE] (yaml exec) built 2 execution wave(s) for 3 command(s)
```

Краткий шаблон — только kind и сообщение:

```sh
githops check pre-commit -v --verbose-template "$k: $m"
```

```
INFO: config loaded from githops.yaml
VERBOSE: resolved 3 command(s)
```

JSON-совместимый шаблон:

```sh
githops sync -v --verbose-template '{"time":"$t","kind":"$k","layer":"$l","msg":"$m"}'
```
