# Визуальный граф

`githops graph` открывает визуальный интерактивный интерфейс для изучения и редактирования конфигурации хуков. Он запускает локальный веб-сервер и открывает браузер автоматически при передаче флага `--open`.

```sh
githops graph --open
```

## Что показывает граф

Вкладка **Flow** отображает направленный граф, где:

- **Узлы хуков** (именованные блоки на верхнем уровне) представляют каждый настроенный Git-хук.
- **Узлы команд** расположены под каждым хуком и показывают все команды этого хука.
- **Узлы ссылок на определения** визуально отличаются и показывают, какие команды являются ссылками на общие определения, а не встроенными командами.
- **Рёбра** соединяют команды с их зависимостями, показывая порядок выполнения.

С первого взгляда видно, какие хуки настроены, какие команды они разделяют через определения и каков порядок выполнения внутри каждого хука.

## Вкладки

### Hooks (Хуки)

Вкладка Hooks показывает все поддерживаемые Git-хуки. Настроенные хуки выделены. Нажмите на любой хук, чтобы открыть панель редактора, где можно:

- Включить или отключить хук.
- Переключить параллельное выполнение.
- Добавлять, переупорядочивать и удалять команды.
- Добавлять встроенные команды или ссылки на определения.
- Редактировать `run`, `depends`, `env` и настройки кэша для каждой команды.

Изменения сохраняются в `githops.yaml` немедленно при сохранении — отдельного шага записи нет.

### Commands (Команды)

Вкладка Commands перечисляет все уникальные встроенные команды во всех настроенных хуках. Это удобно для нахождения дублирующихся команд, которые можно вынести в определения. Здесь можно редактировать значение `run` команды — изменение распространяется на каждый хук, который её использует.

### Definitions (Определения)

Вкладка Definitions показывает все именованные определения. Можно создавать новые, редактировать существующие, изменять тип определения (одиночная команда или список команд) и удалять определения. При удалении определения все записи `$ref`, указывающие на него, автоматически удаляются из всех хуков.

### Flow (Граф)

Вкладка Flow показывает граф зависимостей, описанный выше. Можно:

- **Перемещаться** по графу перетаскиванием холста.
- **Масштабировать** колесом мыши или элементами управления в углу.
- **Перетащить маркер** между двумя узлами команд для добавления ребра зависимости.
- **Выделить ребро и нажать Delete** для удаления зависимости.

Изменения в графе немедленно записываются в `githops.yaml`.

### Cache (Кэш)

Вкладка Cache показывает текущую конфигурацию кэша и все кэшированные записи. Можно:

- Включить или отключить глобальное кэширование.
- Изменить директорию кэша.
- Очистить все записи кэша одним нажатием.
- Видеть каждую кэшированную команду, её ключ кэша (SHA-256) и как давно она была кэшрована.

## Обновление в реальном времени

Интерфейс каждую секунду проверяет изменения в `githops.yaml`. Если отредактировать файл непосредственно в редакторе, пока граф открыт, интерфейс обновится автоматически без перезагрузки страницы.

## Запуск без автооткрытия

```sh
githops graph
```

githops выводит URL в терминал (`http://127.0.0.1:7890` по умолчанию). Откройте его в любом браузере вручную. Если порт 7890 занят, githops выбирает случайный доступный порт.

## Остановка сервера

Нажмите `Ctrl+C` в терминале, где запущен `githops graph`.
